# Notes for a future v3

This file collects breaking changes worth making in the next major version
(v3). They are deferred from v2 to keep the public API stable; batch them into a
single v3 release rather than breaking callers piecemeal.

When you hit something in v2 that you would fix if you didn't care about
compatibility, add it here instead of doing it.

## Rename the Voice Insights `CallEvent`

This library is a single Go package (`twilio`) with short, friendly type names,
so two resources cannot both be called `CallEvent`. Two exist:

- `CallEvent` (`voice_insights_events.go`) — the Voice Insights call-events
  resource (`edge`, `group`, `level`, `name`, ...).
- `CallEventLog` (`call_events.go`) — the core 2010 API Call Event resource, a
  log of the request/response pairs Twilio exchanged during a call.

The core-API resource is the more natural owner of the plain name `CallEvent`,
but the Insights type shipped first, so v2 keeps it and names the newer one
`CallEventLog`.

For comparison, the official OpenAPI-generated `twilio-go` sidesteps the clash
with one Go package per API version (`rest/api/v2010` vs `rest/insights/v1`) and
prefixed names like `ApiV2010CallEvent`. This library deliberately trades that
for a single flat package, and this rename is the cost of that choice.

In v3:

- Rename the Voice Insights type `CallEvent` -> `InsightsCallEvent` (and, if it
  reads better, `CallEventsService` / `CallEventsPage` / `CallEventsPageIterator`
  to match).
- Rename `CallEventLog` -> `CallEvent` so the core 2010 resource owns the name.
